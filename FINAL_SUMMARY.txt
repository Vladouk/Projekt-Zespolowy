OPTION 1: Windows Batch Files (Easiest)
─────────────────────────────────────
  1. Double-click: backend/start.bat
     → Backend starts on http://localhost:5000
  
  2. Double-click: frontend/start.bat (new terminal)
     → Frontend starts on http://localhost:8000
  
  3. Visit: http://localhost:8000/frontend/start.html
     → Page will auto-detect backend and redirect to app

OPTION 2: Manual Terminal
──────────────────────────
  Terminal 1:
    $ cd backend
    $ python run.py
    → Backend on http://localhost:5000
  
  Terminal 2:
    $ cd frontend
    $ python -m http.server 8000
    → Frontend on http://localhost:8000

OPTION 3: One Command (All-in-One)
────────────────────────────────────
  $ python run_all.py
  → Starts both servers automatically
  → Opens browser to http://localhost:8000

OPTION 4: Automated Setup
───────────────────────────
  $ python setup.py
  → Checks dependencies
  → Installs missing packages
  → Initializes database
  → Provides next steps

PRICING FORMULA:
  Base = $15
  Price = Base × Distance × Category Multiplier × Urgency Multiplier

CATEGORY MULTIPLIERS:
  • Beverages: 1.0x
  • Food: 1.5x
  • Electronics: 2.0x
  • Documents: 0.8x
  • Other: 1.2x

URGENCY OPTIONS:
  • Standard: 1.0x
  • Urgent (2x base price): 2.0x

REVENUE SPLIT:
  • Client pays: 100%
  • Courier gets: 80%
  • Platform takes: 20%

EXAMPLES:
  ┌─────────────────────────────────────────────┐
  │ Order: 2.5km food delivery (not urgent)    │
  │ = $15 × 2.5 × 1.5 × 1.0                    │
  │ = $56.25 (client) → $45 courier, $11.25 us │
  ├─────────────────────────────────────────────┤
  │ Order: 5km urgent electronics               │
  │ = $15 × 5 × 2.0 × 2.0                      │
  │ = $300 (client) → $240 courier, $60 us     │
  └─────────────────────────────────────────────┘

┃                    🔄 ORDER PROCESSING FLOW                                ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

1. CLIENT CREATES ORDER
   ├─ Fills form (address, product, category, urgency)
   ├─ Frontend calculates price dynamically
   ├─ System creates order (status: "new")
   └─ Database logs action

2. COURIER BROWSES AVAILABLE
   ├─ Courier logs in
   ├─ Views nearby orders
   ├─ Distance calculated automatically
   └─ Price shown to courier

3. COURIER ACCEPTS ORDER
   ├─ Clicks "Accept" button
   ├─ Order status changes to "assigned"
   ├─ Courier assigned as order.courier_id
   └─ System logs assignment

4. COURIER NAVIGATES TO CLIENT
   ├─ Courier travels to delivery address
   ├─ GPS location can be tracked (future)
   ├─ Status updates to "in_progress"
   └─ Client can see status

5. DELIVERY COMPLETE
   ├─ Courier marks as "delivered"
   ├─ Completion time recorded
   ├─ Client notified (future: SMS/email)
   └─ Amount locked for payment

6. CLIENT LEAVES REVIEW
   ├─ Client rates courier (1-5 stars)
   ├─ Optional comment added
   ├─ Courier stats updated
   └─ Review visible in system

7. PAYMENT & SETTLEMENT
   ├─ Platform charges commission
   ├─ Courier amount transferred
   ├─ Activity logged
   └─ Cycle complete

BACKEND NOT STARTING
  Solution:
  1. Check Python: python --version
  2. Install deps: pip install -r requirements.txt
  3. Delete old DB: rm backend/database.db
  4. Start: python backend/run.py
  5. Check port 5000 not in use: netstat -an | grep 5000

FRONTEND NOT CONNECTING
  Solution:
  1. Check backend running: curl http://localhost:5000/api/health
  2. Check frontend port 8000 available
  3. Check browser console (F12)
  4. Check localStorage (DevTools → Application)
  5. Check CORS headers in response

LOGIN NOT WORKING
  Solution:
  1. Use credentials: admin / admin123
  2. Check database exists: ls backend/database.db
  3. Check JWT_SECRET_KEY set
  4. Check token in localStorage
  5. Check browser console for errors

DATABASE ISSUES
  Solution:
  1. Delete database.db
  2. Restart backend
  3. Database will auto-initialize
  4. Default admin user will be created

PORT CONFLICTS
  Solution:
  • Backend: Use port 5001 instead of 5000
  • Frontend: Use port 8001 instead of 8000
  • Update configuration in files
