## 🔑 Ключевые концепции

### 1. Умная маршрутизация (Smart Routing)

Когда клиент создает заказ → **автоматически находятся ближайшие курьеры**:

```javascript
// Клиент создает заказ "Пиво на ул. Шевченко, 20"
// ↓
// Server рассчитывает координаты
// ↓
// SELECT all active couriers in 5km radius
// ↓
// Сортирует по расстоянию (ближайший первый)
// ↓
// Отправляет уведомление топ 5 курьерам
// ↓
// Курьер #1 (1.2км) видит первым
// Курьер #2 (2.5км) видит вторым
// Курьер #3 (3.7км) видит третьим
// ... и т.д.
```

### 2. Расчет цены (Pricing Algorithm)

```
Цена = 15 грн/км × расстояние × коэффициент категории × коэффициент срочности

Пример: Пиво 2 бут (категория: beverages = 1.0, срочность: обычная = 1.0)
        Расстояние: 2.5км
        Цена = 15 × 2.5 × 1.0 × 1.0 = 37.50 грн ✅
```

### 3. GPS Трекинг (Location Tracking)

```
Курьер включает app:
  ↓
Каждые 30 секунд отправляет свои координаты
  ↓
Server сохраняет: courier_locations table
  ↓
Client видит курьера на карте в реальном времени
```

## 🧪 Тестовые сценарии

### Сценарий 1: Полный цикл (Happy Path)

```
1. CLIENT логинится
2. CLIENT создает заказ "Пиво 2 бут" на "ул. Шевченко, 20"
3. COURIER #1 видит уведомление (находится в 2.5км)
4. COURIER #1 принимает заказ
5. CLIENT видит "Курьер найден" + рейтинг COURIER #1
6. COURIER #1 обновляет статус: "В пути"
7. CLIENT видит курьера на карте
8. COURIER #1 обновляет статус: "Доставлено"
9. CLIENT получает уведомление "Доставлено"
10. CLIENT оставляет отзыв ⭐⭐⭐⭐⭐

✅ УСПЕШНО!
```

### Сценарий 2: Курьер далеко (Error Case)

```
1. CLIENT создает заказ на ул. Шевченко
2. COURIER #1 находится в 8км (дальше чем 5км)
3. COURIER #1 НЕ видит этот заказ в списке
4. Только COURIER #2 (2км) видит

✅ ВАЛИДАЦИЯ РАБОТАЕТ!
```

### Сценарий 3: Admin видит всё (Admin Dashboard)

```
1. ADMIN логинится
2. ADMIN видит на карте всех 7 курьеров (зеленые точки)
3. ADMIN видит список всех 156 заказов
4. ADMIN может отредактировать цену заказа
5. ADMIN видит статистику: доход, рейтинги, активные доставки

✅ ADMIN КОНТРОЛЬ!
```

## ⚠️ Важные замечания

### Безопасность

- ✅ Курьер видит только заказы в радиусе 5км
- ✅ Клиент видит только свои заказы
- ✅ Admin видит всё (но только если `role == 'admin'`)
- ✅ Все endpoints требуют JWT токена

### Оффлайн функциональность

- ✅ Client может создать заказ offline (сохраняется в IndexedDB)
- ✅ При reconnect заказ синхронизируется с сервером
- ✅ Courier видит cached заказы offline

### Производительность

- ⚡ GPS отправляется раз в 30 секунд (не чаще!)
- ⚡ Поиск курьеров оптимизирован (радиус 5км)
- ⚡ Маршруты кешируются

## 🤔 FAQ

**Q: Как узнать координаты клиента?**
A: Используем `navigator.geolocation.getCurrentPosition()` на клиенте

**Q: Как узнать координаты адреса доставки?**
A: Используем geocoding API (Nominatim, Google Maps)

**Q: Что если курьер offline?**
A: Заказ остается в статусе "new", ждет online курьера

**Q: Как считается расстояние?**
A: Используем Haversine formula (расстояние между двумя точками на земле)

**Q: Может ли один курьер принять несколько заказов одновременно?**
A: В ETAP 2 нет, но можно добавить в ETAP 3
